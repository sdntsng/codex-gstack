# Project Retro

Use this skill to understand what actually happened over a recent work period.

## Workflow
1. Define the time window or commit range.
2. Gather execution evidence.
   - commits
   - major files touched
   - test activity
   - bug-fix churn
   - shipping patterns
3. Summarize what moved forward.
4. Identify quality or execution concerns.
5. End with practical improvements for the next cycle.

## Good outputs
Return:
- what shipped
- where effort clustered
- what quality signals improved or degraded
- biggest wins
- biggest risks
- specific habits or follow-ups for the next period

## Guardrails
- Base claims on observable evidence.
- Avoid vague morale language.
- Prefer operational lessons over generic encouragement.
