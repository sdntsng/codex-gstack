# Design Review Live

Use this skill after the UI exists.

## Workflow
1. Open the implemented experience in a real browser.
2. Audit the highest-signal dimensions.
   - hierarchy
   - layout rhythm
   - typography
   - color and contrast
   - interaction states
   - responsiveness
   - product feel
3. Separate issues by type.
   - objective breakage
   - usability issues
   - visual inconsistency
   - generic or low-intent design choices
4. Prioritize the fixes that most improve clarity and perceived quality.
5. If asked to implement, keep fixes surgical and verify them visually.

## Output expectations
Return:
- audit summary
- findings by severity or impact
- screenshots or evidence when useful
- recommended fixes in priority order

## Guardrails
- Do not nitpick everything equally.
- Focus on changes that materially improve the experience.
- Respect the existing design system when one already exists.
